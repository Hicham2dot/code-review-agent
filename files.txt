MODÈLES PARTAGÉS (utilisés par tous les modules)
----------------------------------------------------------------

internal/models/types.go
  - Issue        : un problème détecté (ID, type, sévérité, localisation, message, suggestion, confiance, source)
  - Location     : position dans le fichier (file, start_line, end_line)
  - AnalysisResult : résultat complet d'une analyse (timestamp, hash du diff, liste d'Issues, Summary, durée)
  - Summary      : agrégat statistique (nb critical/major/minor, qualité globale, confiance moyenne)
  - DiffHunk     : un bloc de diff parsé (fichier, lignes ajoutées/supprimées, contexte)
  - Importé par : parser, analyzer (local + llm), formatter, cache

----------------------------------------------------------------
PIPELINE D'ANALYSE
----------------------------------------------------------------

  input.diff
      │
      ▼
  internal/parser/diff.go        ← ParseDiff(content string) → []DiffHunk
      │  lit le format unified diff et extrait les blocs modifiés
      ▼
  internal/analyzer/local.go     ← LocalAnalyze(code string) → []Issue
      │  détecte les patterns dangereux sans IA (SQL injection, secrets, etc.)
      │
  internal/analyzer/llm.go       ← LLMAnalyze(code string, provider string) → []Issue
      │  envoie le diff à Claude / GPT-4 / Ollama et retourne des Issues
      ▼
  internal/formatter/
      ├── json.go      FormatJSON(results)     → string JSON
      ├── markdown.go  FormatMarkdown(results) → string Markdown
      └── cli.go       FormatCLI(results)      → string coloré pour terminal
      ▼
  stdout / fichier de sortie

STATUT : toutes ces fonctions sont des stubs (retournent []/{}/nil)

  - Get(hash string) → interface{}   : récupère une analyse depuis le cache
  - Set(hash, data)  → error         : stocke une analyse dans .cache/ (JSON)
  - Clear()          → error         : vide tout le cache
  - Évite de re-analyser deux fois le même diff (clé = hash SHA du contenu)

internal/config/config.go

      LLMConfig      : provider (claude/gpt/ollama), model, max_tokens, temperature
      CacheConfig    : enabled, dir, TTL

      OutputConfig   : format (json/markdown/cli)
  - LoadConfig() lit les variables d'env et .code-review-agent.yml
  - STATUT : stub (retourne &Config{})

INFRASTRUCTURE
----------------------------------------------------------------

Dockerfile
  - Stage 1 (builder) : golang:1.21-alpine → compile le binaire avec "go build ./cmd"
  - Stage 2 (runtime) : alpine:latest      → copie seulement le binaire (~50 MB final)
  - ENTRYPOINT : ./code-review-agent

docker-compose.yml
  - Service ollama         : image ollama/ollama, port 11434, volume ollama_data
  - Service code-review-agent : build local, dépend de ollama, LLM_PROVIDER=ollama

Makefile
  - make build        → scripts/build.sh      (go build -o code-review-agent ./cmd)
  - make test         → scripts/test.sh       (go test ./...)
  - make run          → build + ./code-review-agent
  - make docker-build → scripts/docker_build.sh
  - make clean        → supprime le binaire et .cache/
